# SaaS OS — CEO Memory

## Mission
Construire le **WordPress du SaaS** : une plateforme modulaire, simple à installer et simple à administrer, permettant de créer rapidement des applications métiers avec frontend + backend + modules activables.

## Direction produit actuelle
- Le point fort n’est pas juste la modularité technique.
- Le vrai wedge = **blueprints métier + wizard d’installation ultra simple + admin type WordPress**.
- Vision : vendre l’**usine à pelles** pendant la ruée vers le SaaS.

## Thèse produit
Pour chaque métier :
1. comprendre le problème métier réel,
2. proposer un frontend adapté,
3. activer une combinaison de modules,
4. ajouter un module si le besoin n’est pas couvert,
5. enrichir progressivement une bibliothèque réutilisable.

## Priorités CEO
1. Wizard ultra simple
2. Sélecteur / générateur de blueprints métier convaincant
3. Architecture modules solide et compréhensible
4. Premiers métiers démontrables de bout en bout
5. Simplicité d’installation/admin au niveau WordPress
6. Dimension **agent / IA native** : produit installable, paramétrable et extensible facilement par Claude ou un autre agent

## Axe agent / IA
- SaaS OS ne doit pas être seulement no-code / low-code pour humains.
- Il doit aussi être **agent-friendly** : un agent doit pouvoir comprendre rapidement la structure, choisir un blueprint, activer des modules, configurer une instance et ajouter un module manquant sans friction inutile.
- La doc, les conventions de modules, les blueprints et le wizard doivent donc être pensés aussi comme une **interface pour agents**.

- Toujours chercher la combinaison la plus vendable, la plus simple et la plus réutilisable.
